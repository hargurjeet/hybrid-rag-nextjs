# Frontend — Claude Context (Next.js)

## Stack

- **Next.js 16.2.6** — App Router, no `src/` directory (paths directly under `frontend/`)
- **React 19.2.4**
- **TypeScript** — strict mode, path alias `@/*` maps to `frontend/`
- **Tailwind CSS v4** — uses `@import "tailwindcss"` in globals.css; **no `tailwind.config.js`**
- **shadcn/ui** — component library on top of Radix UI primitives
- **Inter** (Google Fonts) as web fallback for SF Pro system font

## Directory Structure

```

│   ├── layout.tsx          ← root layout (fonts, metadata, ThemeProvider)
│   ├── page.tsx            ← root page (tab routing: Ask / Evaluation)

│   ├── layout/             ← NavBar, Sidebar, TabNav

├── lib/
│   ├── api.ts              ← queryRAG(), streamEvaluation() fetch wrappers

├── types/
│   └── rag.ts              ← QueryRequest, QueryResponse, DocumentResult

├── .env.local              ← NEXT_PUBLIC_API_URL=http://localhost:8000

├── package.json
└── tsconfig.json
```

## Key Conventions

- **No `src/` directory** — imports use `@/components/...`, `@/lib/...`, `@/types/...`
- **Tailwind v4** — utility classes work as normal; for custom tokens use CSS variables defined in `globals.css`; do NOT create `tailwind.config.js`
- **Server vs Client components** — default to Server Components; add `"use client"` only when using hooks (useState, useEffect, event handlers)
- **shadcn/ui components** live in `components/ui/` — generated by CLI, do not manually edit them
- **Design system** — all colors, radii, shadows use CSS variables from `globals.css`; never hardcode hex values in components

```css
/* Use these CSS variables in components */

```

## Running

```bash
cd frontend
npm install
npm run dev     # http://localhost:3000
npm run build   # production build check
npm run lint    # ESLint
```

Backend must be running on port 8000: `uv run uvicorn api.main:app --port 8000 --reload`

## API Integration

All backend calls go through `lib/api.ts`. The base URL comes from `process.env.NEXT_PUBLIC_API_URL` (set in `.env.local`). Never call `fetch()` directly in components — always use the wrappers in `lib/api.ts`.

## Important Notes

- This is Next.js 16 — APIs may differ from your training data. The `AGENTS.md` file has a note about this.
- `next.config.ts` uses TypeScript (not `.js`)
- ESLint config is in `eslint.config.mjs` (flat config format, ESLint v9)
